1. Application Architecture

1.1 Technology Stack

Backend

  • Flask - Python web framework
  • SQLAlchemy - ORM for database operations
  • Flask-Login - User authentication
  • Flask-SocketIO - Real-time messaging
  • Africa's Talking - SMS API for notifications

Frontend

  • Jinja2 - Template engine
  • Tailwind CSS - Utility-first CSS framework
  • Alpine.js - Lightweight JavaScript framework
  • Font Awesome - Icon library
  • Socket.IO - Client-side real-time communication

1.2 Database Schema

User
├── id (PK)
├── email
├── password_hash
├── phone_number
├── is_verified
├── role (client/personnel)
├── created_at
├── referral_code
└── referred_by (FK -> User.id)

Job
├── id (PK)
├── title
├── description
├── service_type
├── duration_days
├── budget
├── status (open/in_progress/completed)
├── client_id (FK -> User.id)
├── personnel_id (FK -> User.id)
└── created_at

Bid
├── id (PK)
├── job_id (FK -> Job.id)
├── personnel_id (FK -> User.id)
├── amount
├── message
├── estimated_days
├── status (pending/accepted/rejected)
└── created_at

Message
├── id (PK)
├── sender_id (FK -> User.id)
├── receiver_id (FK -> User.id)
├── job_id (FK -> Job.id)
├── content
├── attachment_url
├── is_read
└── created_at

Review
├── id (PK)
├── job_id (FK -> Job.id)
├── reviewer_id (FK -> User.id)
├── reviewee_id (FK -> User.id)
├── rating
├── comment
└── created_at

Referral
├── id (PK)
├── referrer_id (FK -> User.id)
├── referred_id (FK -> User.id)
├── created_at
└── reward_given

2. API Endpoints

2.1 Authentication Endpoints

Endpoint Method Description Authentication
/login GET, POST User login None
/register GET, POST User registration None
/verify-otp/<user_id> GET, POST OTP verification None
/logout GET User logout Required

2.2 Job Endpoints

Endpoint Method Description Authentication
/jobs GET List all jobs None
/jobs/create GET, POST Create a new job Required (Client)
/jobs/<job_id> GET View job details None
/jobs/<job_id>/bid GET, POST Place a bid on a job Required (Personnel)
/jobs/<job_id>/accept-bid/<bid_id> POST Accept a bid Required (Job Owner)
/jobs/<job_id>/complete POST Mark job as completed Required (Job Owner)
/jobs/<job_id>/review GET, POST Leave a review Required (Job Owner or Personnel)

2.3 Messaging Endpoints

Endpoint Method Description Authentication
/messages GET List all conversations Required
/messages/<user_id> GET, POST View/send messages with a user Required
/messages/job/<job_id>/<user_id> GET Start conversation about a job Required (Job Owner or Personnel)

2.4 Admin Endpoints

Endpoint Method Description Authentication
/admin GET Admin dashboard Admin only
/admin/users GET List all users Admin only
/admin/users/<user_id> GET View user details Admin only
/admin/jobs GET List all jobs Admin only
/admin/jobs/<job_id> GET View job details Admin only
/admin/message/<user_id> GET, POST Send message to user Admin only
/admin/revoke-access/<user_id> POST Revoke user access Admin only
/admin/referrals GET List all referrals Admin only

3. User Guide

3.1 For Clients

Posting a Job

  1. Navigate to your dashboard and click "Post a Job" button
  2. Fill in the job details form with:
    • Job title
    • Detailed description
    • Service type
    • Expected duration
    • Budget
  3. Submit the form to publish your job

Managing Bids

  1. View all bids on your job posting
  2. Review bid details including:
    • Bid amount
    • Estimated completion time
    • Personnel's message
  3. Message personnel for clarification if needed
  4. Accept the most suitable bid

Completing Jobs

  1. Once the job is finished, mark it as complete
  2. Leave a review for the personnel
  3. Rate their work on a scale of 1-5
  4. Provide detailed feedback in your review

3.2 For Construction Personnel

Finding Jobs

  1. Browse available jobs on the jobs page
  2. Filter jobs by service type to find relevant opportunities
  3. View job details to understand requirements

Placing Bids

  1. Click "Place Bid" on jobs you're interested in
  2. Provide your bid details:
    • Bid amount
    • Estimated completion time
    • Message to client explaining your approach
  3. Submit your bid

Managing Accepted Jobs

  1. Track your accepted bids on your dashboard
  2. Communicate with clients through the messaging system
  3. Update clients on progress
  4. After job completion, leave a review for the client

4. Admin Guide

User Management

  • View Users - Access the complete list of users and their details
  • User Details - View comprehensive information about each user
  • Message Users - Send SMS notifications to users
  • Revoke Access - Disable user accounts when necessary

Job Management

  • View Jobs - Monitor all jobs posted on the platform
  • Job Details - Access detailed information about each job
  • Manage Bids - Review all bids placed on jobs
  • Flag/Delete Jobs - Remove inappropriate job listings

Referral Management

  • Track Referrals - Monitor all referrals on the platform
  • Manage Rewards - Process referral rewards
  • View Statistics - Access referral program performance metrics

System Monitoring

  • Dashboard - View key platform metrics and statistics
  • System Status - Monitor the health of platform components
  • Activity Logs - Track important user actions
